# Profile: `default` — theory / linear-spiral

The safe default. A course that names no profile gets this one. It suits concept-first material and
is a defensible starting point for almost any subject, because it changes the *least* about the
core.

## Configuration

| Dimension | Setting |
| :--- | :--- |
| **Macro spine** | **Linear with a spiral overlay** — units run in dependency order, and the few enduring ideas are deliberately revisited at least once more, later, in a different context and at higher difficulty. |
| **Entry point** | **Theory-first** — each unit explains the idea, then applies it. Lessons still *frame* with a concrete problem (the arc's first move is never skipped); the difference is that the explanation follows immediately rather than being withheld until the learner has struggled. |
| **Checkpoint placement & frequency** | **One advisory checkpoint at the end of each unit**, plus one mid-course consolidation for courses longer than ~6 units. Worded as a recommendation, never as a gate. |
| **Unit granularity** | **Medium** — a unit is one coherent idea cluster, roughly 3–6 lessons. Small enough to finish in a sitting or two; large enough that the spiral has something to return to. |

## How the spiral is planned

Do this at syllabus time, not by accident:

1. From the outcomes (skill `backward-design`), mark the **few enduring ones** — the ones worth
   remembering long after the course.
2. For each, plan a **first pass** where it is introduced and a **later revisit** where it returns
   in a new context, at a higher difficulty level (apply → analyze, analyze → create).
3. Space the revisits: put at least one unit of other material between a pass and its revisit.
4. Write the revisit into the syllabus explicitly. A revisit that is not planned reads as
   repetition, not as a spiral.

Supporting outcomes are taught once, in dependency order, and are not spiralled — spiralling
everything is just repetition.

## Advisory checkpoint — the wording contract

A checkpoint is a short, self-checkable consolidation the learner is *invited* to do:

> **Recommended checkpoint.** Before moving on, work through these — they consolidate the unit and
> the next unit assumes them. If any is uncomfortable, revisit <lesson> first.
>
> 1. <a retrieval prompt over the unit's enduring idea>
> 2. <one applied task using the running example>
> 3. <one item that connects this unit to the previous one>

Rules: it is **advisory**, it is **self-checkable** (every item has a reference answer or a stated
criterion), and it never withholds the next unit. Static content cannot enforce progression and
must not pretend to.

## Best fit / weak fit

- **Best fit** — concept and theory-heavy subjects, foundational courses with strict prerequisites,
  and any course whose shape is not yet known (this is the low-risk choice).
- **Weak fit** — subjects taught best from an authentic task or case first, and courses organized
  around demonstrated competencies rather than content units. Those are what the later-increment
  profiles are for; until one ships, `default` still produces a coherent course.

## Core invariants this profile reuses unchanged

Backward-design backbone · canonical lesson arc · formative-feedback loops (including the
consistency check) · the rubric's core layer. This profile **reconfigures none of them** — see
`profiles/README.md`.
