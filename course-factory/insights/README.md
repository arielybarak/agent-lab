# insights/ — the cross-course knowledge digest

**Knowledge, not tooling.** This is the factory's shared, cross-course lessons-learned store
(constitution Principle XII, § Structural Constraints). It is a sibling of `pedagogy/`, but distinct:
`pedagogy/` is a curated teaching-technique library; `insights/` accumulates *what this factory
learned building its own courses*.

## It starts empty — on purpose

There is **no pre-existing corpus** here. Nothing is seeded from the reference course
`System_Design_SelfLearn` (an unvalidated idea-source, never inherited by authority). The digest
grows **only** from this factory's own harvested courses, going forward. **An empty or missing digest
is a valid input everywhere it is read** — it is never a blocker.

## Read rule (who consumes it)

- **Intake** — spec 001 (FR-025): read as an input to the clarify interview + brief authoring.
- **Syllabus compose** — spec 002.
- **Skeleton / lesson drafting** — spec 003.

Each reader treats an empty digest as valid input.

## Write rule (how it grows) — user-invoked only

Two paths, **both user-invoked**, with **no automatic or scheduled trigger** (settled 2026-07-08,
spec 004 FR-015/016): a per-insight capture (the user tells an agent to log an insight here), and a
`setup-retro`-style bulk harvest that pumps a course's `FEEDBACK.md` up into here on demand.
Harvesting is **owned by spec 004** and never fires on course completion or a phase transition by
itself. Spec 001 only **reads** this directory; it does not write it.
