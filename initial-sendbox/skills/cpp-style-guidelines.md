# Skill: C++ Style and Simplicity

**Trigger:** When generating, reviewing, or refactoring C++ code.

**Directives:**
1. **Standard:** Follow the [Google C++ Style Guide](https://google.github.io/styleguide/cppguide.html).
2. **Student-Centric Modifications:**
   - Prefer claer, standard library solutions.
   - Avoid overly clever template metaprogramming or complex macro definitions.
   - Use clear, descriptive variable names (fits to conventions).
   - Ensure inputs and outputs to functions are clearly documented.
   - Prefer structured programming and basic classes over deep inheritance hierarchies (unless it impacts code quality).
