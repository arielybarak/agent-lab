# Skill: Add Educational Comments

**Trigger:** When asked to "add educational comments", explain code, or write tutorials.

**Directives:**
1. Do not just state *what* the code does (e.g., "iterates over array").
2. Explain the *why* (e.g., "We iterate to find the maximum value to normalize the rest of the dataset").
3. Explain the *how* for complex operations (e.g., "This bitwise shift multiplies the number by 2 efficiently at the hardware level").
4. Pitch the explanation to a Computer Engineering student: assume moderate programming knowledge (threads, processes) but explain domain-specific, mathematical, or algorithmic nuances clearly.
5. Use block comments for function-level explanations and inline comments for tricky or dense algorithmic lines.
